def runAegisIndefinitely (p : Expr) : MetaM Unit := do
  -- 1. Create a dummy IORef initialized to false
  -- We use liftM because IO.mkRef is an IO action
  let dummySignal ← liftM (IO.mkRef false : IO (IO.Ref Bool))

  -- 2. Call the prover with the dummy signal
  -- Since nothing in this scope flips dummySignal to true, 
  -- the recursion in 'find' will never trigger the 'return none' branch.
  let result ← Aegis.proveOrDisprove p dummySignal

  -- 3. Handle the result
  match result with
  | some res => 
    if res.status then
      IO.println s!"Proof found: {res.proof}"
    else
      IO.println s!"Refutation found: {res.proof}"
  | none => 
    -- This branch is technically unreachable with a dummySignal
    IO.println "Prover was unexpectedly interrupted."
Why this works for "Truly Running Forever"
Isolation: The dummySignal is local to this call. No other thread or "Harness" has a handle to this specific IORef, so dummySignal.set true can never be called.

The Check is "Transparent": Inside Aegis/Prover.lean, the line if ← (liftM stopSignal.get : MetaM Bool) then return none will execute at the start of every depth, find that the value is still false, and immediately proceed to the next search step.

Minimal Overhead: Checking a boolean reference at the start of each deepening level is computationally negligible compared to the universalSearch itself.
